# Тестовое задание на базе DummyJSON API

## 1. Опыт ручного тестирования
Да.

## 2. Опыт нагрузочного тестирования
Умею работать с JMeter.

## 3. Опыт тестирования безопасности
Только базовые проверки, типа SQL-инъекций.

## 4. Инструменты, использованные в задании
Python, pytest, fixtures, parametrize, pydantic, allure.

## 5. Команда для запуска тестов

Активировать venv:

```bash
pip install -r requirements.txt
python -m pytest --alluredir=allure-results
allure serve allure-results
```

Команда `allure serve allure-results` используется для просмотра результатов.

> Allure-степы добавлены только в `test_user_login` - это весьма затратно по времени, добавил их там, чтобы показать, что умею ими пользоваться.

## 6. Краткий список реализованных проверок

- **Авторизация** - успешный логин, неверный логин/пароль, null/пустой username, null/пустой password
- **Данные пользователя** - валидный токен, просроченный токен, без токена
- **Создание корзины** - один/несколько продуктов, пустой список, null/невалидный userId
- **Получение корзины по id** - валидный id, None/пустой/невалидный id
- **Получение корзины по userId** - валидный userId, None, невалидный userId
- **Обновление корзины** - один/несколько/пустой список продуктов × merge on/off, None/невалидный cart_id, невалидные типы quantity и productId
- **Удаление корзины** - валидный id, None/пустой/невалидный id

## Обнаруженные баги и причины падения некоторых тестов

1. **Несоответствие модели продукта** - `POST /carts/add` возвращает `discountedPrice`, а `GET /carts/{id}` - `discountedTotal`. Тест на получение корзины по id падает на валидации модели.
2. **DELETE/GET cart с пустым id** - API отдаёт список всех корзин вместо 404.
3. **Ложноположительные результаты при update** - невалидные `quantity` (str/null) и `productId` (str/null) не вызывают ошибку на стороне API.
